The Challenge

1. The Challenge

Finding the needle in the haystack — across 10 to 100s of dense corporate reports per asset.

Every copper property in your pipeline generates a trail of annual reports, quarterly releases, technical studies, and regulatory filings. The critical details — a revised recovery rate, a footnote redefining reserve categories, a guidance change buried in an operational review — are scattered across dense data tables and unstructured text. Each report contains 50+ tables, each table 50+ semantically unique values with specific context about the property, unit, time period, and process.

Data aggregators tried to solve this. They extract, standardise, and sell access. But standardisation breaks traceability — you cannot trace a value back to its source. Granular detail and footnotes are lost. Values go stale. The result: "You cannot trust the data, so you go back to the source or call an analyst."

AI and LLMs offer a new path — but without deep context, they produce assumptions, not answers. An LLM cannot reliably extract and classify 250,000+ unique values into clean time series. It understands headlines, not data.

The root cause is the same in both cases: the missing link between a reported value and its source.

Our Solution

2. How We Solve It

The Source Value Key — the missing link between reported data and AI-powered answers.

We build the Source Value Key: a unique identifier that ties every extracted value to its exact source — report, page, table, row, and context. It is the foundation that makes AI reasoning trustworthy and data fully traceable.

First-in-Class Data Extraction

Tabular data is extracted using a deterministic, in-house algorithm — not an LLM. Every value is QC'ed and assigned a Source Value Key at the point of extraction. This is where traceability begins: each number carries its origin from day one.

Document Hierarchy Context

The Source Value Key doesn't stop at the table cell. Every paragraph, section header, and footnote is linked into the document hierarchy — so each value carries the full context of what it relates to. Nothing is lost in standardisation.

Context Data: LLM Interface

AlfaRev AI — our question-and-answer interface — reasons over data indexed by Source Value Key. When it answers a question, it cites the exact source — and distinguishes between fully keyed data (traceable) and uncontextualised data (use with caution). Built on the strongest reasoning models available today.

Peer Comparison: Built In

Because every value carries its Source Value Key, data across assets is contextualised and aligned at source — units, timescales, and reporting conventions are resolved before the data reaches you. That makes peer comparison native: compare revenue, reserves, production, or costs across assets by country, mining method, or commodity — without the manual work of exporting, cleaning, and aligning spreadsheets.

The result: ask a question, get a sourced answer — with every value traceable via its Source Value Key back to the exact report, page, and table it came from. Compare one asset or twenty — the data is already aligned.

Q&A

7. Questions & Answers

Where are you in the product's development?

We have developed the hardest part — the technology for data extraction and classification that produces the Source Value Key. We have started testing AlfaRev AI and are targeting an MVP launch in June 2026. But building the right product requires real-world feedback from professionals who work with this data every day — that's why the Partner Program exists. We need Partner support to get the traction and direction to deliver something that genuinely works for the people who use it.

How can I trust the AI's responses?

The AI cites data sources for every reply. Users can choose whether it should only consider data with a Source Value Key or include uncontextualised data. The AI uses AlfaRev's APIs to work exclusively on QC'ed and contextualised data, with full transparency and traceability.

Which data types are provided?

Revenue & Costs, Resources & Reserves, and Production (tonnes, grade, recovery). Additional reported data without a Source Value Key is still accessible to the AI but should be used with caution, as it lacks full contextualisation.

How far back does the historical data go?

We aim to ingest the full reporting history for each asset. At minimum, we cover 10 years of historical data where available. The depth depends on what the company has published, but partners can request priority on specific time periods.

How many sources are ingested per asset?

At minimum, annual reports — but we aim to cover quarterly reports, annual reports, and technical/operational studies. This means not only the number of assets on the platform grows over time, but also the depth of sources per asset.

How fast can you add more assets?

Our technology has been tested on mining reports published over the past 15 years. Due to the lack of reporting standards, new datasets require ongoing development — but the Partner Program includes 2 monthly ingestions to continuously expand your coverage.

Can I upload my own PDFs and run the extraction myself?

No. The extraction and classification engine is proprietary technology that we operate on our side. Mining reports vary widely in structure, formatting, and conventions — there is no "drop a PDF and get clean data" shortcut. Every new asset goes through our pipeline and is QC'ed before delivery. That QC step is what produces the Source Value Key and ensures the data is trustworthy. What you receive is the finished product: clean, contextualised, traceable data — not a tool you need to operate yourself.

Can I compare assets against their peers?

Yes — peer comparison is built in. Because every value is contextualised with its Source Value Key, data across assets is already aligned at source: units, currencies, timescales, and reporting conventions are resolved during extraction. You can compare revenue, reserves, production, grade, or costs across assets — filtered by country, mining method, commodity, or any combination. No manual export, cleaning, or unit conversion required.

Can you extract drilling data (collars, intersections, coordinates)?

Drilling results are structured tabular data — collar coordinates, dip/azimuth, end-of-hole depth, and economic intersections. Our extraction engine ingests these tables, and AlfaRev AI can reason over them: query intersections, compare holes, and surface context like geographic reference systems or local grid definitions that are often buried in footnotes.

However, drilling data is not part of the standard time-series delivery (Revenue, Reserves, Production). This is exactly where the Partner Program matters: as a Partner, you directly influence if and how new data types like drilling results are formalised into structured, QC'ed deliverables with a Source Value Key. Until then, AlfaRev AI still returns results from ingested drilling tables — just without the full contextualisation of a keyed time series.

How realistic is the projected platform growth?

The projected 390+ assets by end of Year 2 is a scenario, not a guarantee. It depends on our ability to onboard new customers and expand commodity coverage. What is guaranteed: your 42 Partner assets and unlimited access to everything else on the platform. If we scale faster, you benefit. If we scale slower, your Partner assets and cost are unaffected.

What is AlfaRev's competitive edge?

Greenfield mining data extraction — AI-native with no legacy systems. Full source traceability via the Source Value Key, deterministic extraction (not probabilistic), and deep context that enables AI reasoning at a level not seen before.
